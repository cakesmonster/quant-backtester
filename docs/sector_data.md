# 同花顺板块映射数据

## 数据文件

| 文件 | 类型 | 格式 | 说明 |
|------|------|------|------|
| `data/cache/raw_sectors.json` | **倒排索引** | `{板块名: [股票代码列表]}` | 原始数据，465 板块 → 成分股 |
| `data/cache/stock_sector_map.json` | **正排索引** | `{股票代码: {行业: [...], 概念: [...]}}` | 个股 → 所属板块 |

### 正排格式示例

```json
{
  "000723": {
    "行业": ["煤炭开采加工"],
    "概念": ["煤化工概念"]
  },
  "002579": {
    "行业": ["元件"],
    "概念": ["5G", "6G概念", "AI PC", "AI手机", "AI眼镜", ...]
  }
}
```

### 倒排格式示例

```json
{
  "煤化工概念": ["600989", "601015", "000723", "600740", ...],
  "光伏概念": ["601012", "688599", "300274", ...],
  "半导体": ["688981", "002371", "603501", ...]
}
```

## 数据来源

同花顺（10jqka.com.cn），覆盖：

- **概念板块**：375 个（例：光伏概念、机器人概念、人工智能...）
- **行业板块**：90 个（例：半导体、白酒、汽车整车...）
- **覆盖股票**：~2000 只（A 股活跃交易标的）
- **总映射**：~61,000 条

## 队友查找逻辑

```
个股代码（如 000723）
    │
    ▼ 正排查
{行业: ["煤炭开采加工"], 概念: ["煤化工概念"]}
    │
    ▼ 倒排查（取所有板块的成分股并集）
煤化工概念 → ["600989", "601015", "000723", ...]
煤炭开采加工 → ["601898", "600985", "000723", ...]
    │
    ▼ 去重 + 遍历分时图
计算 Pearson r 两两互相关 → 找队友
```

## 更新数据

数据非实时（板块成分股变动频率低），建议 **每月更新一次**。

```bash
cd /root/.hermes/projects/sundial
python scripts/batch_fetch_sectors.py    # 分批拉取，约 30 分钟
python scripts/format_map.py             # 格式化为正排
```

### 拉取脚本说明

`batch_fetch_sectors.py`：
- 375 个概念板块分 5 批，90 个行业分 2 批
- 每批之间冷却 3 分钟，防止 IP 被同花顺限流
- 每页请求刷新 v_code Cookie（同花顺反爬）
- 翻页上限 100 页（跳过融资融券等无区分度的大板块）
- 支持断点续传（已有数据不重拉）
- 产出 `raw_sectors.json`

`format_map.py`：
- 加载 `raw_sectors.json`
- 结合 akshare 板块分类，区隔"概念"和"行业"
- 构建正排索引 `stock_sector_map.json`

## 注意事项

- **核心标的不在自身概念中**：同花顺概念板块不包含命名标的本身。例：比亚迪(002594)不在"比亚迪概念"中，宁德时代(300750)不在"宁德时代概念"中——这是同花顺的设计逻辑
- **概念板块有区分度差异**："融资融券"(3790只) vs "AI眼镜"(~190只)，大板块边际信息量低
- **建议查队友时过滤大板块**：概念成分股 > 500 只的跳过，优先用细分概念
