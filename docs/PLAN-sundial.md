# 日晷 (Sundial) — 实施计划

> A股市场望远镜 · 观日影之移，知战机之瞬

---

## 一、项目定位

**日晷 = 市场数据看板**，专注于公开市场数据的聚合展示，不包含任何买卖策略逻辑。

- 已有能力来自：同花顺 API、mootdx 通达信 Level-2
- 数据全部公开，无策略泄露风险
- 独立部署，不依赖 board_monitor / quant-backtester

---

## 二、技术选型

| 层 | 选型 | 说明 |
|----|------|------|
| Web 框架 | FastAPI | 与现有系统一致 |
| 前端 | 单页 HTML + Plotly.js + htmx | 零前端构建 |
| 数据源 | mootdx (Level-2/逐笔/分钟K) + 同花顺 API | 免费 |
| 时序存储 | Parquet | 日线K线本地缓存（3300只 × 10年） |
| 元数据存储 | SQLite | 涨停历史 / 股票基础信息 |
| 部署 | systemd + 公网开关 | 同现有模式 |

## 三、功能清单

### P0 — 核心展示（第一版必须）

#### 3.1 连板天梯

```
展示所有涨停股，按连板数排列：

首板 | 60xxxx 某某股份 [芯片] +10.0%
     | 00xxxx 某某科技 [新能源] +9.98%

2连板 | 60xxxx 某龙头 [军工] +10.0%

3连板 | ...
```

- 每只涨停股展示：代码、名称、连板数、所属板块
- **点击/Hover** → 弹出面板：相关概念、市值、流通市值、市盈率、换手率、量比
- **昨日涨停今日破板** → 灰色标记「淘汰」，显示今日涨跌幅

**数据来源：**
- 涨停判断：当日 `close == limit_up_price`
- 连板数据：SQLite 历史表，每日增量更新
- 板块/市值/PE：同花顺 API

#### 3.2 市场情绪仪表盘

页面顶部一排核心数据卡片：

| 指标 | 计算方式 |
|------|----------|
| 涨停家数 | 当日涨停总数 |
| 跌停家数 | 当日跌停总数 |
| 炸板家数 | 触及涨停但未封住的 |
| 炸板率 | 炸板 / (涨停+炸板) |
| 连板晋级率 | N连板晋级N+1连板的比例 |
| 昨日涨停今日均收益 | 所有昨涨停今日平均涨幅 |

**数据来源：** 从连板数据 SQLite 衍生计算。

#### 3.3 昨日涨停今日表现

单独一个模块：

```
昨日涨停 28 只，今日：
  继续涨停 5 只  ████████░░░░░░░
  红盘    12 只  ██████████████████████░░░░
  绿盘     8 只  ██████████░░░░░░
  跌停     3 只  ███░░░░░░░░

平均涨幅 +2.3%
最高 +10.0%（某某股份）
最低 -10.0%（某某科技）
```

→ 短线情绪核心先行指标，判断今日打板胜率。

#### 3.4 三时段热榜

三个时间点自动快照：

| 时间 | 说明 |
|------|------|
| 11:30 | 上午收盘热榜 |
| 15:00 | 下午收盘热榜 |
| 21:00 | 同花顺一小时热榜 |

每份热榜展示：排名、代码、名称、热度值、概念标签、涨停状态。

→ 点击热榜里的涨停股 → 右侧展示该股分时图 + 找队友推荐。

**数据来源：** `eq.10jqka.com.cn/open/api/hot_list/v1/hot_stock/a/hour/data.txt`

### P1 — 重要功能

#### 3.5 涨停股分时走势图

点击任意涨停股 → 弹出/侧边展示当日 1 分钟 K 线分时图（Plotly）。

**数据来源：** mootdx `get_bars(frequency=7)` 拉 1-min K 线，实时拉取不持久化。当天数据盘中缓存内存、收盘丢弃。

#### 3.6 大单买入卖出

展示当日大单异动：

```
时间     股票       方向    数量      金额
09:35   600xxx    买入    5000手    1200万
10:12   000xxx    卖出    3000手     580万
```

**数据来源：** mootdx 逐笔成交 `get_transaction_data()`，筛选 `vol ≥ 3000手`。

#### 3.7 模拟账户持仓盈亏

一个独立页面：

- 当前持仓（代码、名称、数量、成本、现价、盈亏%）
- 今日成交记录
- 总资产/可用资金/总盈亏

**数据来源：** 同花顺模拟炒股 API。`.env` 中配置 USRID/dep，不提交 GitHub。

#### 3.8 找队友 + 推荐

点击任一只涨停股 → 计算并展示：

- **同概念股票**（从热榜 API 的 `concept_tag` 直接取）
- **走势相似股票**（1-min K 线 Pearson 相关系数 > 0.7）
- 展示每个队友的：名称、相关性、当前涨幅、分时小图

→ 展示同概念和走势相似的股票，公开市场数据，不涉及策略判断。

### P2 — 增强功能

#### 3.9 竞价异动

9:15-9:25 集合竞价阶段：

| 信号 | 条件 |
|------|------|
| 诱多陷阱 | 涨停价挂单量骤降 50%+ |
| 弱转强 | 跌停拉到翻红 |
| 竞价爆量 | 竞量 > 昨日总成交 5% |

**数据来源：** mootdx 集合竞价数据，09:25 一次性拉取。

#### 3.10 板块攻击路线

时间轴展示当日资金轮动：

```
09:30 → 芯片冲高（涨停 3 家，龙头 xxx）
09:45 → 新能源接力（涨停 2 家，龙头 xxx）
10:15 → 军工轮动（涨停 1 家）
```

**数据来源：** 所有涨停股按首次触板时间分组，按板块聚合。

#### 3.11 换手率/量比异常标记

在连板天梯中：
- 换手率 > 20% → 🔴 标红（死亡换手）
- 量比 > 3 → ⚡ 标记（异常放量）

**数据来源：** 同花顺 API 直接返回。

## 四、数据存储设计

### SQLite 核心表

```sql
-- 涨停历史（每日增量）
CREATE TABLE limit_up_history (
    date TEXT, code TEXT, name TEXT,
    board_count INTEGER,        -- 连板数
    first_time TEXT,            -- 首次触板时间
    last_time TEXT,             -- 最后封板时间
    is_broken BOOLEAN,          -- 是否炸板
    sector TEXT,                -- 所属板块
    PRIMARY KEY (date, code)
);

-- 热榜快照（三时段：11:30 / 15:00 / 21:00）
CREATE TABLE hot_rank_snapshot (
    date TEXT, slot TEXT,        -- slot: "1130" / "1500" / "2100"
    rank INTEGER, code TEXT, name TEXT,
    heat_value REAL,             -- 热度值
    concept_tag TEXT,            -- 概念标签
    is_limit_up BOOLEAN,         -- 是否涨停
    change_pct REAL,             -- 涨跌幅
    PRIMARY KEY (date, slot, code)
);
```

股票基础信息（市值/PE/换手率/量比）不持久化，点击时实时拉取：
- **东财 API**：`push2.eastmoney.com/api/qt/stock/get` → 市值/流通/PE/换手率/量比
- **概念标签**：同花顺热榜 API 已有 `concept_tag`
```

### 热榜快照

三时段数据存入 SQLite，支持按日期/时段回看历史热榜。
